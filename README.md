### What I learned

This project helped me understand several new concepts:

- **HTML Elements**
  - `<time>` — Wrapping dates with the `datetime` attribute for semantic meaning
  - `<figure>` — Grouping images with captions or related content

- **CSS Concepts**
  - Font shorthand — Combining font properties (weight, size, family) into one line
  - Box-shadow — Creating depth with the offset-x, offset-y, blur, and color values

- **Accessibility**
  - Using the `datetime` attribute on `<time>` elements to provide machine-readable dates alongside human-friendly text

### Continued development

In future projects, I want to focus on:

- **Adding JavaScript for interactivity** — I want to learn how to make web pages more dynamic and engaging. This includes creating animations, handling user interactions like clicks and form submissions, and building features that respond to user actions in real-time.

- **Deepening accessibility knowledge** — While I've started with semantic HTML and datetime attributes, I want to go further. This means understanding ARIA roles and attributes, improving keyboard navigation for users who don't use a mouse, and learning how to test with screen readers to ensure my projects are truly accessible to everyone.

### Useful resources

- [Figma](https://www.figma.com/) - Used to view and analyze the design mockup before coding, helping me understand the layout and spacing requirements.

- [Google Fonts](https://fonts.google.com/) - Provided the Figtree font family used throughout the project. A great free resource for web-safe fonts.

- [TinyPNG](https://tinypng.com/) - Helped me compress and optimize the images in the project without losing quality, making the page load faster.

- [Cloudinary](https://cloudinary.com/) - Used to host the Open Graph image for social media sharing.

- [Perfect Pixel](https://chrome.google.com/webstore/detail/perfectpixel-by-welldonec/dkaagdgjlophiddqccjgplachon0304v) - Chrome extension that allowed me to overlay the design mockup directly on my live page for pixel-perfect accuracy.

### AI Collaboration

I used **GitHub Copilot** and **Gemini CLI** throughout this project, and they were genuinely helpful. Specifically:

- **Explaining concepts** — When I had questions about things like font shorthand, box-shadow properties, and semantic HTML elements like `<time>` and `<figure>`, Copilot provided clear explanations with examples that made the concepts easier to understand.

- **Documentation help** — Copilot and Gemini CLI helped me organize my thoughts and structure the README, making it clearer and more professional.

The overall experience was positive. The guidance was patient and encouraging, which made learning feel less intimidating. Everything worked smoothly, and I didn't run into any confusing or unclear responses. This is a great way to learn by doing rather than just reading documentation alone.
